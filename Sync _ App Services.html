Development Mode

Development Mode (Dev Mode) allows you to configure data models and queryable fields in your client application.

Dev Mode disables drafts (i.e. changes made in your app deploy immediately).

All collections for this cluster with a defined and valid JSON schema will be synced to devices. Selecting a new cluster to sync restarts the syncing process.

Note: Sync requires an Atlas cluster using MongoDB version 5.0 or greater

When adding new tables in your mobile app, a new collection will be created in MongoDB. Select the database where you want new collections to be created in.

Configure Queryable Fields (Optional)

Select fields from your JSON Schema to construct queries on. In developer mode, queryable fields will be added automatically when they are used by a connected client.

Client Max Offline Time

Setting a maximum offline time allows Device Sync to prune metadata after the maximum offline time has passed, leading to more efficient storage utilization in MongoDB Atlas. Clients which are offline longer than the maximum offline time will experience a client reset.

Client Recovery

Client Recovery allows a client to automatically attempt recovery of its local unsynced data when experiencing a fatal error or client reset. If successful, this will result in the client re-uploading data that was created prior to the reset. Disabling this setting will cause clients to discard local changes upon client reset.
